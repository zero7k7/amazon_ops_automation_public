# 规则优先级说明

本项目的报告建议必须按以下优先级判断，避免搜索词、增强数据或历史反馈覆盖产品级事实。

## 1. 产品级窗口指标优先于搜索词级指标

- 产品级判断统一引用 `product_window_metrics`。
- 1d / 7d / 14d / 30d 的广告点击、花费、广告订单、广告销售额、总订单、总销售额、自然单、ACOS、TACOS 以 `marketplace + asin` 汇总结果为准。
- 单个搜索词或单个 ASIN 定向只能作为广告处理证据，不允许单独上升为“产品整体无单”。
- 如果产品 14 天广告订单 > 0，不得因为某个搜索词 0 单而判为“广告无单”。

## 2. ERP / 广告产品级窗口优先于增强数据

- 广告和 ERP 的产品级窗口是强证据。
- `traffic_sales` 和 `search_query_performance` 是辅助证据，用于解释流量、页面成交、推荐报价和搜索漏斗变化。
- 增强数据周期未知、过期或只作背景参考时，不得触发强动作。
- Search Query Performance 只能作为搜索漏斗辅助证据，不能直接生成否词。

## 3. 人工确认优先于系统猜测

- `autoopt_feedback_input.json` 中的 `manual_root_cause` 和 `manual_action_taken` 优先于系统的 Listing 猜测。
- 同一产品再次触发类似问题时，应优先引用人工确认原因。
- 例如“demo 产品已人工确认页面承接问题，并完成一次价格或 Listing 调整”应进入 `manual_learning_log_YYYYMMDD.json`，后续报告只复查效果，不重复泛化成 Listing 问题。

## 4. 已执行冷却优先于今日动作

- 已确认执行的对象短期内不重复占用 P0 / P1 主动作位。
- 普通已执行动作冷却 3 天。
- 带人工确认原因的动作冷却 7 天。
- 冷却期内对象应进入“明日复查”或“已执行动作复查”，只看效果，不重复推相同动作。

## 5. 成本 / 利润风险不混入广告动作

- `target_acos = 0`、广告前利润 <= 0、成本配置异常，属于财务 / 配置核对。
- 成本或利润未确认前，不建议加预算、不建议放量。
- 利润风险可以进入“利润 / 成本待核对”，但不应自动归为广告无转化、真无单或 Listing 问题。

## 6. 广告处理队列优先级

- 明显不相关词，且 clicks >= 3、orders = 0：可以建议否定精准。
- 核心词即使 0 单，也不直接否词，优先降竞价或观察。
- 相关但转化差，clicks >= 5 或 spend >= 5 且 orders = 0：建议降竞价。
- ASIN 定向 clicks >= 3 或 spend >= 5 且 orders = 0：建议暂停 ASIN 定向或降竞价。
- clicks <= 2 且 orders = 0：只进入 Excel 低优先级明细，不进入 HTML 主动作区。

## 7. 输出分层

- HTML 主页面只放今日需要看的动作、复查和关键证据。
- Excel 保留完整明细、低优先级对象、搜索词证据、指标校验和增强数据状态。
- `latest_analysis.json` 保存最终报告视图快照，方便测试和后续自我优化读取。
